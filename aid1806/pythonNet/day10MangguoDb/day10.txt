MongoDB(芒果数据库)

数据存储阶段：

文件管理阶段：（.txt  .doc  .xls ）

    优点：数据可以长期保存
        数据有一定的格式化规范
        可以大量存储
        使用简单方便
    缺点：数据一致性差
         用户查找修改不方便
         数据冗余

数据库管理阶段：
    优点：数据结构组织化，降低了冗余
        提高了增删改查的效率
        便与扩展，方便程序调试做自动化处理
    缺点：数据库的使用往往专业性较强，相对比较复杂

数据：能够输入到计算机中并被识别处理的信息的集合
数据结构：研究数据集合中数据之间关系的学科
数据库：按照数据结构，存储管理数据的仓库，是在数据管理
系统软件管理和控制下创建在一定介质上的数据集合
数据库管理系统：数据管理软件，用于维护管理数据库
数据库系统：由数据库，数据库管理系统，开发工具等组成的
工具集合

关系型数据库：采用关系模型来组织数据结构的数据库（二维表）

Qracle  DB2 SOLServer  Mysql  Sqlite

    优点：1，容易理解，逻辑类似常见的表格
         2，使用方便，都使用sql语句，而sql语句非常成熟
         3，数据一致性高，冗余度低，完整度高
         4，技术成熟，可以使用外部链接等复杂操作
    缺点：1，每次操作都需要sql语句解析，消耗较大
         2，不能很好的满足并发需求，应对海量数据力不从心
         3，关系型数据库内存存在大量的加锁操作，读写能力
         受到限制
         4，数据不够灵活，有时会导致结构复杂化

非关系型数据库（NoSql） --->not only sql
    优点：1，高并发，读写能力强
         2，扩展性强，使用灵活
         3，弱化了数据结构，降低数据一致性
    缺点：1，通用性差，没有sql 那样统一的操作语句
         2，操作灵活，容易混乱
         3，没有join等复杂操作，版本更新快

    Nosql的使用情况
        1，数据的一致性要求低
        2，处理数据的海量并发
        3，数据库大小不容易确定，需要分布扩展
        4，给定的数据结构建立非关系模型更加容易
    Nosql分类
        1，键值数据库  Redis
        2，列存储数据库  
        3，文档数据库   MongoDB
        4，图形数据库

MongoDB数据库（非关系型数据库 ---》 文档数据库）
    1，由c++ 编写的数据库管理系统
    2，支持丰富的增守删改查功能
    3，支持丰富的数据类型
    4，支持众多的编程语言结构(python php c++ js c#)
    5,在NoSql 中技术相对比较成熟
